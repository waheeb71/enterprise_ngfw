# 🛠️ Enterprise NGFW Automation Scripts

يحتوي هذا المجلد على مجموعة من السكريبتات الجاهزة لأتمتة عمليات (التثبيت، التحديث، إعداد الشبكة، والشهادات) الخاصة بنظام Enterprise NGFW.

## 📁 هيكل المجلدات والسكريبتات

### 1. 🔄 سكريبت التحديث (`update.sh`)
يُستخدم لتحديث النظام (المركز / الخادم الوهمي) بعد كتابتك لأكواد جديدة أو إجراء تعديلات على المشروع.

* **طريقة العمل**:
  1. أوقف خدمة الفايروال مؤقتاً.
  2. يسحب الأكواد الجديدة من المجلد المحلي (افتراضياً `~/enterprise_ngfw`) أو من `Git` وينقلها لمجلد التشغيل `/opt/enterprise-ngfw`.
  3. يحدّث أي مكتبات بايثون جديدة.
  4. ينفذ تحديثات قاعدة البيانات (`alembic`).
  5. يعيد تشغيل الخدمة بالأكواد الجديدة.
* **طريقة الاستخدام**:
  ```bash
  # تأكد من إعطاء الصلاحية أول مرة:
  chmod +x scripts/update.sh
  
  # تشغيل التحديث:
  sudo ./scripts/update.sh
  ```

---

### 2. 🚀 سكريبت التثبيت الأساسي (`install/install.sh`)
يُستخدم لتثبيت النظام من الصفر على خادم جديد (Ubuntu/Debian/Kali أو RHEL/CentOS).

* **طريقة العمل**:
  1. يثبت كافة الحزم الأساسية لنظام التشغيل (مثل `python3`, `libpcap`, `iptables`).
  2. ينشئ بيئة بايثون الوهمية ويقوم بتحميل المتطلبات.
  3. ينشئ خدمة التشغيل التلقائي (`systemd`).
  4. يولّد شهادات الـ SSL الأساسية وينشئ مجلدات النظام في `/opt/enterprise-ngfw` و `/etc/enterprise-ngfw`.
* **طريقة الاستخدام**:
  ```bash
  chmod +x scripts/install/install.sh
  sudo ./scripts/install/install.sh
  ```

---

### 3. 🛡️ سكريبتات الشبكة (`networking/`)
تُستخدم لتوجيه حركة مرور أجهزة الشبكة لكي تمر إجبارياً عبر الجدار الناري ليتم فحصها.

* **تفعيل الوكيل الشفاف (Transparent Proxy)**:
  يقوم بتفعيل الـ `IP Forwarding` والـ `NAT` ويضيف قواعد `iptables` لتحويل منافذ `80` و `443` إلى محرك الفحص الخاص بالنظام.
  * **طريقة الاستخدام**:
    ```bash
    chmod +x scripts/networking/setup-transparent-proxy.sh
    sudo ./scripts/networking/setup-transparent-proxy.sh
    ```

* **إلغاء تفعيل الوكيل الشفاف (Remove Proxy)**:
  يقوم بمسح قواعد التوجيه وإعادة الشبكة لحالتها الطبيعية للالتفاف حول الجدار الناري.
  * **طريقة الاستخدام**:
    ```bash
    chmod +x scripts/networking/remove-transparent-proxy.sh
    sudo ./scripts/networking/remove-transparent-proxy.sh
    ```

---

### 4. 🔐 سكريبتات الشهادات (`certs/`)
* **تصدير شهادة الجذر (`export-ca.sh`)**:
  نظراً لأن الفايروال يقوم باعتراض وفك تشفير مسار HTTPS (SSL Inspection)، يجب تثبيت شهادة الجذر في أجهزة المستخدمين/العملاء لتجنب ظهور رسالة "شهادة غير آمنة" في متصفحاتهم. هذا السكريبت يقوم بتجهيز وتصدير الشهادة ليسهل نقلها.
  * **طريقة الاستخدام**:
    ```bash
    chmod +x scripts/certs/export-ca.sh
    sudo ./scripts/certs/export-ca.sh
    ```

---

## 💡 نصائح هامة
- كافة السكريبتات تتطلب صلاحيات موزع النظام الجذري (Root)، لذا استخدم دائماً `sudo` عند تشغيلها.
- سجلات تشغيل التثبيت والتحديث تُحفظ في المسار `/var/log/enterprise-ngfw/` لمراجعتها في حال حدوث أي خطأ.
